## Plugin Manifest

The AI instruction pack's active dependencies live in `ai-plugins.json` at the
project root. It behaves like `package.json` does for Node:

- Declares the stack, layout, and which roles/guides/guidelines/refactoring
  docs/commands this project depends on
- Every `/new-release`, `/edit-release`, and `/finish-release` verifies that
  every declared path resolves on disk before running
- Use `/verify-plugins` to run that check by itself
- Edit `ai-plugins.json` directly to enable/disable a plugin (`enabled: true`
  or `false`) ‚Äî keep it in sync with the checkboxes in `ai-settings.md`

## Frontend Rules

### Component Limits
- Max **150 lines** per component file
- Extract inline sub-components over **30 lines** to their own file in a subfolder adjacent to the parent
- No JSX ternary component-selectors in the return body ‚Äî resolve to a `const` before `return` or use early returns
- When a component has more than 3 handlers or any handler with branching logic, extract to a `use[Name]Controller` hook

### Tailwind CSS
- Max **3 Tailwind utilities** per `className`
- If more are needed, extract a semantic class in `index.css` using `@apply` inside `@layer components`
- Name semantic classes by purpose, not appearance

### TypeScript
- Strict mode ‚Äî no `any`, no type assertions without comment justification
- All business logic in custom hooks, not inline in JSX
- Named predicates for complex conditions

### Frontend Testing
- Vitest + React Testing Library
- Test behavior from the consumer's perspective, not implementation details
- Prefer `getByRole`/`getByLabelText` over `getByTestId`
- New extracted components need their own tests

## Refactoring

### Backend
Trigger a dedicated pass when:
- Route handlers accumulate service logic
- Worker orchestration grows tangled or duplicates stage logic
- Provider logic leaks outside its adapter
- Validation starts doing extraction work, or vice versa

### Frontend
Trigger a dedicated pass when:
- A component exceeds 150 lines
- An inline sub-component exceeds 30 lines
- A `className` has more than 3 Tailwind utilities
- JSX return body contains ternaries that select between components
- Inline handlers with branching logic pile up in a component

### Process
Identify the rule violation, batch by file/stage, change one unit at a time, re-run tests/verification after each. Keep refactors separate from feature work.
